【仕様書】商品在庫管理システム（Ruby on Rails版）

■ 概要
倉庫担当者が使う商品の在庫管理画面を作成する。
在庫数が一定以下になった商品を一覧で強調表示する。

■ 使用技術
- Ruby on Rails
- ActiveRecord
- ERB
- SQLite または MySQL
- RESTful routing

■ テーブル設計
テーブル名: products

- id
- code: string 必須 ユニーク
- name: string 必須
- category: string 必須
- price: integer 必須 デフォルト0
- stock: integer 必須 デフォルト0
- alert_threshold: integer 必須 デフォルト10
- is_active: boolean デフォルトtrue
- description: text nullable
- created_at
- updated_at

■ Migration
productsテーブルを作成する。

カラム:
- code:string
- name:string
- category:string
- price:integer default: 0, null: false
- stock:integer default: 0, null: false
- alert_threshold:integer default: 10, null: false
- is_active:boolean default: true, null: false
- description:text

制約:
- code に unique index を付与する
- code / name / category は null: false
- price / stock / alert_threshold / is_active は null: false

■ Model
Model名: Product

バリデーション:
- code
  - presence: true
  - length maximum: 50
  - uniqueness: true

- name
  - presence: true
  - length maximum: 255

- category
  - presence: true
  - inclusion:
    - 食品
    - 日用品
    - 電化製品
    - 衣類
    - その他

- price
  - presence: true
  - numericality:
    - only_integer: true
    - greater_than_or_equal_to: 0

- stock
  - presence: true
  - numericality:
    - only_integer: true
    - greater_than_or_equal_to: 0

- alert_threshold
  - presence: true
  - numericality:
    - only_integer: true
    - greater_than_or_equal_to: 0

- is_active
  - inclusion: [true, false]

- description
  - allow_blank: true

定数:
CATEGORIES = ["食品", "日用品", "電化製品", "衣類", "その他"]

メソッド:
- low_stock?
  stock <= alert_threshold の場合 true

- status_label
  is_active が false の場合「停止中」
  stock <= alert_threshold の場合「要補充」
  それ以外は「正常」

■ Controller
Controller名: ProductsController

アクション:
- index
- show
- new
- create
- edit
- update
- destroy

Controller追加要件:
- 一覧画面では category パラメータがある場合、where で絞り込みを行う
- is_active が未チェックの場合でも false として保存されるようにする
- Strong Parameters を使用する

index処理:
- Product.all を取得
- params[:category] が present? の場合
  Product.where(category: params[:category])
- 一覧は created_at の降順で表示する

create / update:
- product_params を使って保存する
- 保存成功時は詳細画面へリダイレクト
- 保存失敗時は new または edit を再表示する

■ Routing
config/routes.rb

resources :products
root "products#index"

■ Laravelとの対応

Laravel:
Model: Product
Controller: ProductController
Request: StoreProductRequest / UpdateProductRequest
Route: Route::resource('products', ProductController::class)
View: resources/views/products/

Rails:
Model: Product
Controller: ProductsController
Validation: Productモデル内に記述
Route: resources :products
View: app/views/products/

■ 補足
Railsでは Laravel の StoreProductRequest / UpdateProductRequest のような専用Requestクラスは基本的に使わない。
代わりに以下で対応する。

- バリデーション → Productモデル
- 入力許可 → ProductsController の strong parameters
- 保存処理 → ProductsController の create / update
